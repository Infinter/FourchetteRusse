- (event) Intent: create. Projet fun. Soirée hebdo (vendredi) entre amis; galère récurrente pour décider du repas (fast food / cuisiner / autre).
- (decision) Forme: bot WhatsApp + interface web. Coeur: gérer un roulement de qui décide/régale, historiser pour stats afin d'éviter le 'c'est tout le temps moi'.
- (decision) Boucle coeur: le roulement designe LE DECIDEUR de la semaine -> il propose un repas (fast food/cuisine/autre) -> les autres acceptent ou refusent -> si refus, le refuseur doit contre-proposer. Mecanisme de consensus, pas simple roue.
- (decision) FRONTIERE: la validation (accepter/refuser/contre-proposer) se fait HORS appli, dans la vraie vie / le chat WhatsApp du groupe. L'appli ne gere PAS la nego. Coeur recentre: designer le decideur (roulement) + enregistrer qui a decide & le choix final + stats/historique.
- (decision) Groupe = 4 personnes (petit, intime -> design leger, pas d'admin lourde). Bot WhatsApp interactif: annonce qui a ete roll + commande type /qui pour declencher un roll. Web = dashboard/historique/stats.
- (decision) Mode de travail: coaching LIGHT. Tirage = ALEATOIRE (coup de de via /qui), l'appli ne force PAS l'equite. /stat expose qui a ete roll trop souvent -> les potes s'auto-equilibrent en se charriant. Paradoxe assume: c'est pour rigoler. AME du produit: pretexte a vanner ses potes avec des chiffres a l'appui, pas un outil de justice.
- (decision) Coeur narratif (resume/probleme/solution/boucle/pour-qui/hors-scope) valide par l'utilisateur ('oui ca ressemble bien').
- (decision) Feature confirmee: /quoi enregistre le repas mange. Besoin de REGROUPEMENT intelligent: 'pates bolo' et 'pates carbo' -> categorie 'pates'. Stats par categorie (ex: 8x pates, 5x burger), pas par libelle exact.
- (decision) Vision: usage PUREMENT PERSO (le groupe) d'abord; ouvrir a d'autres groupes plus tard mais TRES secondaire. Ne pas surinvestir la section vision.
- (decision) Criteres de succes valides (4): (1) utilise chaque vendredi via /qui; (2) le debat 'c'est tout le temps moi' s'eteint ou devient une vanne; (3) on decide plus vite; (4) ca fait rire, /stat devient running gag. Pas de seuil chiffre (trop serieux pour un projet fun).
